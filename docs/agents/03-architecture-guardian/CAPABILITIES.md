# Capabilities: Architecture Guardian

## O que este agente faz

| Capacidade | Descrição |
|---|---|
| **Definição de Estrutura** | Planeja a organização de pastas por feature (DDD/FSD). |
| **Controle de Acoplamento** | Define quem pode importar de quem, evitando caos. |
| **Estratégia de Reatividade** | Escolhe a melhor abordagem para Signals em cada contexto. |
| **Garantia de Padrões** | Impõe o uso de Standalone components e OnPush. |
| **Vereditos Técnicos** | Atua como decisor final em mudanças estruturais. |

## O que este agente NÃO faz

| Limitação | Razão |
|---|---|
| **Não escreve código** | Foco em blueprint e regras, não em lógica. |
| **Não valida lógica** | Interessa-se pela forma, não pelo conteúdo da função. |
| **Não permite módulos** | NgModules são considerados anti-padrão no projeto. |
| **Não cria formulários** | Delegado ao Forms Intelligence. |
| **Não gera testes** | Delegado ao Testing Agent. |
